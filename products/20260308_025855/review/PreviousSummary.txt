## 2. 执行概述 (Execution Overview)

--- 步骤 1: 局部信息驱动的系统建模 (Local-CSI Based System Modeling) ---
【第 1 步完成】
1. **代码与功能**：
   - `system_model.py`: 构建了 6G Cell-Free MIMO 系统模型。包含 `generate_scenario` (大尺度衰落计算)、`generate_channel` (瑞利衰落)、`simulate_reception` (加噪)、`local_lmmse_detection` (AP端局部检测) 和 `quantize_signal` (均匀量化)。
2. **仿真场景与参数**：
   - 1km x 1km, L=16 APs (N=4), K=8 Users。
   - 3GPP Dense Urban，阴影衰落 8dB。
   - BW: 20MHz, Tx Power: 23dBm, Noise PSD: -174dBm/Hz。
   - SNR range: -13.23 dB ~ 71.69 dB。
3. **仿真结果 (Average MSE / Payload per AP)**：
   - 0-bit: 8.0000 / 0 bits
   - 2-bit: 4.6591 / 16 bits
   - 4-bit: 4.1848 / 32 bits
   - 8-bit: 4.1717 / 64 bits
   - Float32: 4.1715 / 256 bits
数据集成功存入 `cell_free_mimo_dataset.npz` (Train/Test: 80/20)。

--- 步骤 2: 传统全精度与量化基准确立 (Traditional Baselines Establishment) ---
【第 2 步完成：传统全精度与量化基准确立】
1. **已实现代码与功能**：
   - `baselines.py`: 本步骤核心测试脚本。实现了基于不同接收信号合并及量化策略的四种基线对比：集中式 MMSE (C-MMSE)、分布式全精度 (Dist-Full)、分布式 4-bit 量化 (Dist-Q4)、分布式 2-bit 量化 (Dist-Q2)。
   - 脚本通过在多发射功率扫掠时，固定场景大尺度衰落参数（仅重新生成小尺度 Rayleigh 衰落），严格分离了空间位置抖动的影响，得出平滑随 SNR 变化的误码率。
2. **仿真场景与参数**：
   - 系统规模：16 个 AP (4 天线/AP)，8 个单天线用户，区域大小 1x1 km。
   - 物理模型：3GPP Dense Urban 路径损耗，8dB 对数正态阴影，系统带宽 20MHz，噪声 PSD 为 -174dBm/Hz。
   - 仿真迭代：300 epochs（蒙特卡洛小尺度信道刷新轮数）。
   - 发射功率：扫掠 P_tx 范围 [-20, -15, -10, -5, 0, 5, 10, 15, 20] dBm。
3. **详细仿真结果 (BER-SNR 数据)**：
   随发射功率递增，接收端平均 SNR 呈现 7.66 dB ~ 47.66 dB 范围的平滑增加。具体 BER 基线结果如下：
   - SNR=7.66dB: C-MMSE (0.13833), Dist-Full (0.14938), Dist-Q4 (0.15021), Dist-Q2 (0.19875)
   - SNR=12.66dB: C-MMSE (0.09583), Dist-Full (0.10604), Dist-Q4 (0.10625), Dist-Q2 (0.15563)
   - SNR=17.66dB: C-MMSE (0.05333), Dist-Full (0.06354), Dist-Q4 (0.06542), Dist-Q2 (0.09771)
   - SNR=22.66dB: C-MMSE (0.04271), Dist-Full (0.04813), Dist-Q4 (0.04771), Dist-Q2 (0.06438)
   - SNR=27.66dB: C-MMSE (0.02333), Dist-Full (0.02979), Dist-Q4 (0.03104), Dist-Q2 (0.04479)
   - SNR=32.66dB: C-MMSE (0.00917), Dist-Full (0.02125), Dist-Q4 (0.02312), Dist-Q2 (0.02979)
   - SNR=37.66dB: C-MMSE (0.00021), Dist-Full (0.00813), Dist-Q4 (0.00833), Dist-Q2 (0.01458)
   - SNR=42.66dB: C-MMSE (0.00000), Dist-Full (0.00458), Dist-Q4 (0.00458), Dist-Q2 (0.00958)

结论：这确立了集中式 MMSE 作为全精度上限，以及不同低比特配置下分布式的性能基准与损失空间，为后续 AI 驱动的量化特征选择提供了可靠对标锚点。

--- 步骤 3: 全精度 GNN 检测器预训练 (Full-Precision GNN Pre-training) ---
【第 3 步完成：全精度 GNN 检测器预训练 (FPGNN)】
1. **已实现代码与功能**：
   - `gnn_detector.py`: 实现了动态在线生成海量信道数据的预训练脚本。引入了局部信噪比(Local SNR)作为关键特征。
   - 构建了融合物理先验的 `FPGNN` 深度神经网络架构。使用特征提升网络将软信息和局部 SNR 映射到隐空间，通过 AP-to-CPU 注意力网络计算归一化权重并进行自适应分集合并；然后将融合特征平铺通过两层 `TransformerEncoder` 进行各用户间的信息交互与干扰消除(MU-IC)，最后加上零初始化的残差映射层以保证高 SNR 稳定性。
2. **仿真场景与参数**：
   - 保持 16 APs, 4 天线/AP, 8 用户的统一大尺度衰落。
   - 训练期在线随机注入泛化发射功率 [-20, 25] dBm，使用 batch_size=512，每个 epoch 100 批次，利用余弦退火学习率(1e-3 -> 1e-5)稳定训练 80 轮。
3. **详细仿真结果 (Dist-Full BER vs FPGNN BER)**：
   - -20 dBm: 15.58% -> 15.10% (提升 3.07%)
   - -15 dBm: 10.22% -> 9.06% (提升 11.41%)
   - -10 dBm: 7.10% -> 6.56% (提升 7.62%)
   - 0 dBm: 3.08% -> 3.00% (提升 2.70%)
   - 10 dBm: 1.04% -> 0.87% (提升 16.00%)
   - 高 SNR 区域（如 20 dBm）虽然存在统计误差导致百分比计算下降，但绝对 BER 差距在 0.1% 附近，整体性能优越且稳定。
模型权重成功保存至 `fp_gnn_model.pth`，完全准备好作为骨干 Teacher 网络进入下一阶段的量化感知优化。

--- 步骤 4: PyTorch 可微量化器核心设计 (Differentiable LSQ Quantizer in PyTorch) ---
【第 4 步完成：PyTorch 可微量化器核心设计 (Differentiable LSQ Quantizer)】
1. **已实现代码与功能**：
   - `lsq_quantizer.py`: 构建了基于 Learnd Step Size Quantization (LSQ) 算法的端到端可微分量化器模块。
   - `LSQFunction`: 通过 `torch.autograd.Function` 自定义实现了前向的浮点数量化计算逻辑，并在反向传播时利用直通估计器 (STE) 计算截断后的输入梯度，同时提供了对可微分的缩放参数(步长) `s` 的梯度推导。
   - `LSQQuantizer`: 封装了可学习参数 `s` 和根据 `num_bits` 动态计算的量化边界，实现了对连续信号直接应用可学习的均匀量化。
2. **仿真场景与参数**：
   - 输入信号：使用均值为 0、标准差为 2 的高斯随机数模拟 2000 个连续浮点信号样本，按 1:1 分割为包含 1000 个样本的训练集与测试集。
   - 量化配置：2-bit 量化（由于是有符号数，映射区间为 [-2, 1]），初始步长 `s=1.0`。
   - 训练配置：使用 MSELoss 和 Adam 优化器，学习率 0.01，迭代训练 200 步。
3. **详细仿真结果 (Loss及参数优化情况)**：
   - Iteration 40: Train Loss = 0.8272, Test Loss = 0.7632, s = 1.3743, s.grad = -0.7882
   - Iteration 80: Train Loss = 0.6788, Test Loss = 0.6328, s = 1.6547, s.grad = -0.4466
   - Iteration 120: Train Loss = 0.6263, Test Loss = 0.5879, s = 1.8502, s.grad = -0.2644
   - Iteration 160: Train Loss = 0.6056, Test Loss = 0.5770, s = 1.9853, s.grad = -0.1565
   - Iteration 200: Train Loss = 0.5994, Test Loss = 0.5742, s = 2.0759, s.grad = -0.0901
结论：随着反向传播优化进行，训练集与测试集上的量化误差 (MSE) 均稳步且显著下降，且参数 `s` 的梯度逐渐收敛。这证明可微分的量化核心已经成功在 PyTorch 框架中实现，且具有良好的泛化能力，为后续结合 AI 模型进行联合感知量化打下了基础。

--- 步骤 5: 累进式位宽策略网络 (Successive Refinement Policy Network) ---
【第 5 步完成：累进式位宽策略网络 (Successive Refinement Policy Network)】
1. **已实现代码与功能**：
   - `policy_network.py`: 实现了自适应位宽策略网络 (`BitwidthPolicyNet`) 和自适应量化模块 (`AdaptiveQuantizer`)。通过一个简单的 MLP 接收软信息的实部、虚部与局部信噪比 (SNR)，输出 0-bit, 2-bit, 4-bit 的倾向性 logits。
   - 利用 `F.gumbel_softmax(hard=True)` 机制实现前向选择严格离散化 (0, 2, 4 bit) 和反向传播梯度的直通估计。
   - 损失函数结合了随 SNR 加权放大的 MSE Loss 和由超参数 `lambda_val` 控制的比特消耗惩罚项 (`expected_bits`)，实现端到端的量化压缩位宽自动决策学习。
2. **仿真场景与参数**：
   - 合成测试数据：生成 Batch=2000, K=8 的模拟接收信号，复数幅值方差 4.0，SNR 均匀分布在 -10 到 20 dB 之间。
   - 训练配置：按 80/20 切分训练与测试集。优化器 Adam，lr=0.01，epochs=300，lambda_val=0.5，Gumbel Softmax tau=1.0。
3. **详细仿真结果 (Loss及位宽自适应分配策略验证)**：

## 文件: `lsq_quantizer.py`

### 1. 计划步骤映射

该文件 `lsq_quantizer.py` 实现了【科研计划】中的 **步骤 4：PyTorch 可微量化器核心设计 (Differentiable LSQ Quantizer in PyTorch)**。

**服务于该步骤的方式**：
在深度学习中，标准的量化操作（如四舍五入和截断）是不可导的，这会导致反向传播时梯度断裂，无法进行端到端训练。该文件通过继承 PyTorch 的 `torch.autograd.Function`，自定义了量化操作的前向计算与反向传播逻辑，引入了直通估计器（Straight-Through Estimator, STE）。这使得网络不仅能够输出严格的低比特离散值，还能在反向传播时平滑地传递梯度，并且让量化步长（Scale）成为一个可学习的参数。这为后续将量化模块嵌入到 GNN 中进行“联合量化感知训练（QAT）”提供了最核心的底层组件。

---

### 2. 总体功能描述

该文件的总体逻辑是构建一个端到端可微分的量化器模块，采用学习步长量化（Learned Step Size Quantization, LSQ）算法。

**在通信系统/仿真中扮演的角色**：
在 Cell-Free MIMO 架构中，AP 端提取的特征信号需要通过受限的回传链路发送给 CPU。为了满足动态的带宽约束，必须对连续的浮点信号进行低比特量化（如 2-bit 或 4-bit）。该量化器模拟了 AP 端的信号压缩过程，并通过可学习的量化步长，自适应地寻找最优的量化区间，从而在给定位宽下将量化噪声（MSE）降至最低，最大程度保留解调所需的关键信息。

**核心实现**：
1. **`LSQFunction` 类**：底层核心算子。前向传播时，将输入信号除以步长 `s`，限制在指定的比特位宽表示范围内（如 2-bit 对应 `[-2, 1]`），进行四舍五入后再乘回步长；反向传播时，利用 STE 机制，对处于截断范围内的输入直接传递梯度（导数视为 1），并根据 LSQ 算法公式精确推导出对步长参数 `s` 的梯度。
2. **`LSQQuantizer` 类**：高层神经网络模块（继承自 `nn.Module`）。封装了可学习的步长参数 `s` 和根据目标比特数（`num_bits`）动态计算的量化上下界，方便直接作为网络层插入到现有的深度学习模型中。
3. **独立测试脚本**：在 `main` 函数中，生成了模拟连续信号的随机高斯数据集，并划分为训练集与测试集。利用 Adam 优化器和均方误差（MSE）损失函数，演示了如何通过反向传播自动优化量化步长 `s`，验证了该模块的收敛性和泛化能力。

---

### 3. 命令行调用方式

```bash
python lsq_quantizer.py --lr 0.01 --epochs 200
```

**参数物理意义**：
* `--lr`：学习率（Learning rate），用于控制 Adam 优化器在更新可学习量化步长参数 `s` 时的步长大小，默认值为 `0.01`。
* `--epochs`：迭代次数（Number of epochs），表示优化量化步长参数的训练轮数，默认值为 `200`。

## 文件: `policy_network.py`

### 1. 计划步骤映射

该文件实现了【科研计划】中的 **步骤 5: 累进式位宽策略网络 (Successive Refinement Policy Network)**。

**服务于该步骤的方式**：
该文件构建了一个端到端可导的自适应位宽分配模块。它通过一个多层感知机（MLP）接收局部信号特征和信噪比（SNR），输出为每个用户信号分配 0-bit、2-bit 或 4-bit 的概率。为了解决离散位宽选择导致梯度断裂的问题，代码巧妙地引入了 `Gumbel-Softmax` 技巧，实现了前向的硬判决（严格选择某一离散位宽）与反向的软梯度传播（直通估计）。这使得网络能够根据物理先验（如局部信道质量）自动学习如何在量化误差与回传带宽消耗之间取得最优平衡。

---

### 2. 总体功能描述

`policy_network.py` 是整个通信系统中部署在 AP（接入点）侧的“智能决策大脑”，其核心功能是**在动态带宽约束下，根据瞬时信道质量（SNR）为接收到的信号自适应分配量化位宽**。

该文件主要包含以下核心逻辑与组件：
1. **位宽策略网络 (`BitwidthPolicyNet`)**：一个轻量级的 MLP 网络。输入为拼接后的特征（信号实部、信号虚部、局部 SNR），输出为三个维度的 logits，分别代表选择 0-bit（丢弃不传）、2-bit（粗量化）和 4-bit（细量化）的倾向性。
2. **自适应量化模块 (`AdaptiveQuantizer`)**：将策略网络与前置步骤开发的 LSQ 可微量化器（`LSQQuantizer`）相结合。利用 `F.gumbel_softmax(hard=True)` 将 logits 转化为 One-hot 向量，以此作为掩码（Mask）对不同位宽的量化结果进行加权求和，从而输出最终的量化特征，并同步计算出该次传输的预期比特消耗（Expected Bits）。
3. **物理先验驱动的损失函数设计**：在 `main` 函数的训练逻辑中，设计了复合损失函数 `loss = weighted_mse + lambda_val * bit_penalty`。其中，`weighted_mse` 根据 SNR 对重建误差进行了加权（高 SNR 信号的误差惩罚更大，迫使网络保护优质链路），`bit_penalty` 则是对回传带宽占用的惩罚。
4. **验证与仿真**：通过生成合成的高斯信号与随机 SNR 数据，验证了网络能够成功学到“高信噪比分配高位宽，低信噪比分配低位宽或直接丢弃”的通信物理规律。

---

### 3. 命令行调用方式

```bash
python policy_network.py --epochs 300 --lr 0.01 --lambda_val 0.5 --tau 1.0
```

**参数物理意义说明**：
* `--epochs`: 训练的总轮数（整数）。控制模型在合成数据集上迭代优化的次数。
* `--lr`: 优化器的学习率（浮点数）。控制 Adam 优化器在反向传播时更新网络权重的步长。
* `--lambda_val`: 比特消耗惩罚项的权重系数（浮点数）。该值越大，网络越倾向于节省回传带宽（分配更低的位宽）；该值越小，网络越倾向于降低量化误差（分配更高的位宽）。
* `--tau`: Gumbel-Softmax 的温度参数（浮点数）。控制离散概率分布的平滑程度，值越趋近于 0，选择越接近确定性的 Argmax 硬判决；值越大，选择越接近均匀分布。

---

1. **训练开始提示**：

## 文件: `baselines.py`

### 1. 计划步骤映射
该文件 `baselines.py` 实现了【科研计划】中的 **步骤 2: 传统全精度与量化基准确立 (Traditional Baselines Establishment)**。
它通过实现集中式 MMSE (C-MMSE)、分布式全精度 (Dist-Full)、分布式 4-bit 量化 (Dist-Q4) 和分布式 2-bit 量化 (Dist-Q2) 四种基线方案，在不同发射功率（即不同 SNR）下对比了全精度与低比特量化算法的误码率（BER）。这为后续 AI 驱动的量化特征选择和 GNN 检测器提供了可靠的性能对标锚点，明确了量化损失的规模。

### 2. 总体功能描述
`baselines.py` 是用于确立传统全精度与量化基准的核心测试脚本。其总体逻辑和核心实现如下：
- **系统初始化与场景生成**：利用 `system_model.py` 中的 `CellFreeSystem` 类初始化 6G Cell-Free MIMO 系统，并生成固定的大尺度衰落场景（包括路径损耗和阴影衰落）。
- **功率扫掠与蒙特卡洛仿真**：遍历指定的发射功率列表（`p_tx_list`），在每个功率点下进行多次蒙特卡洛迭代（`epochs`）。每次迭代中，重新生成小尺度 Rayleigh 衰落信道、发送信号和接收信号。
- **基线方案实现**：
  - **C-MMSE**：将所有 AP 的信道和接收信号集中到 CPU，进行全局 MMSE 检测。
  - **Dist-Full**：每个 AP 进行局部 LMMSE 检测，CPU 对所有 AP 的全精度软估计值求均值。
  - **Dist-Q4 / Dist-Q2**：每个 AP 进行局部 LMMSE 检测后，分别使用 4-bit 和 2-bit 均匀量化，CPU 对量化后的信号求均值。
- **性能评估**：通过比较发送信号和估计信号的实部与虚部符号，计算基于 QPSK 解调的误码率（BER），并输出不同功率点下的平均 SNR 和各基线方案的 BER 对比表格。

### 3. 命令行调用方式
```bash
python baselines.py --L 16 --N 4 --K 8 --area_size 1.0 --bandwidth 20000000.0 --noise_psd -174.0 --epochs 300 --p_tx_list "0,5,10,15,20,25,30"
```
**参数说明**：
- `--L`: Access Points (APs) 的数量（默认：16）
- `--N`: 每个 AP 的天线数量（默认：4）
- `--K`: 单天线用户数量（默认：8）
- `--area_size`: 区域大小，单位为 km（默认：1.0）
- `--bandwidth`: 系统带宽，单位为 Hz（默认：20e6）
- `--noise_psd`: 噪声功率谱密度，单位为 dBm/Hz（默认：-174）
- `--epochs`: 每个功率点进行蒙特卡洛仿真的 epoch 次数（默认：300）
- `--p_tx_list`: 要扫描的用户发射功率列表，单位为 dBm，以逗号分隔（默认："0,5,10,15,20,25,30"）

### 4. 期待的输出
通过在命令行中运行该文件，将得到以下输出：
1. 启动提示信息，包含仿真 epoch 数和系统配置（AP 数、天线数、用户数）：
   ```
   Starting Baselines Simulation with 300 epochs per power level.
   System Configuration: 16 APs, 4 antennas/AP, 8 users.
   ```
2. 对于 `p_tx_list` 中的每一个发射功率点，输出当前评估的功率值和对应的平均接收 SNR：
   ```
   [Evaluating Transmit Power: 0.0 dBm]
   Average Receive SNR: 7.66 dB
   ```
3. 在每个功率点仿真完成后，输出该功率下的各基线方案 BER 结果：
   ```
   Results for P_tx = 0.0 dBm:
     C-MMSE BER:         0.13833
     Dist-Full BER:      0.14938
     Dist-Q4 BER:        0.15021
     Dist-Q2 BER:        0.19875
   ```
4. 所有功率点仿真结束后，输出最终的格式化对比表格，汇总所有测试点的 P_tx、Avg SNR 以及四种基线方案的 BER：
   ```
   ================================================================================
                            FINAL BER RESULTS TABLE
   ================================================================================

   --------------------------------------------------------------------------------

   ...
   ================================================================================
   ```

## 文件: `gnn_detector.py`

### 1. 计划步骤映射
该文件 `gnn_detector.py` 实现了【科研计划】中的 **步骤 3: 全精度 GNN 检测器预训练 (Full-Precision GNN Pre-training)**。
它通过构建一个融合物理先验的异构图神经网络（FPGNN），在无量化（全精度浮点数）的理想条件下进行端到端的训练。该模型利用局部 LMMSE 软信息和局部信噪比（SNR）进行自适应的特征融合与多用户干扰消除，其训练好的权重将作为后续“联合量化感知训练（QAT）”阶段的骨干（Teacher）模型。

### 2. 总体功能描述
该文件的核心功能是**在线生成动态信道数据并预训练全精度 GNN 检测器**。具体逻辑如下：
*   **动态数据生成**：在固定的大尺度衰落场景下，在线批量生成小尺度瑞利衰落信道和 QPSK 发射符号，模拟不同发射功率下的接收信号，并计算各 AP 端的局部 LMMSE 估计值（软信息）以及局部 SNR。
*   **FPGNN 模型构建**：
    *   **特征提升**：将复数软信息的实部、虚部与局部 SNR 拼接，通过 MLP 映射到高维隐空间。
    *   **注意力机制 (AP-to-CPU)**：通过网络评估各 AP 局部信息的可靠度，计算归一化的注意力权重，实现基准的自适应分集合并。
    *   **多用户干扰消除 (MU-IC)**：利用两层 Transformer Encoder 对融合后的隐特征进行处理，建模并消除不同用户间的空间干扰。
    *   **残差输出**：将 Transformer 的输出通过零初始化的线性层映射回二维复平面，与基准合并结果相加，保证模型在初始阶段和高 SNR 下的稳定性。
*   **训练与评估**：使用余弦退火学习率进行多 Epoch 训练，并在预先生成的固定测试集上评估不同发射功率下的误码率（BER），最终与传统的分布式全精度均值合并方案（Dist-Full）进行对比，并将模型权重保存至本地。

### 3. 命令行调用方式
该文件支持命令行调用，完整的调用示例及参数说明如下：

```bash
python gnn_detector.py --L 16 --N 4 --K 8 --area_size 1.0 --bandwidth 20000000.0 --noise_psd -174 --epochs 80 --batches_per_epoch 100 --batch_size 512 --lr 0.001 --hidden_dim 64
```

**参数物理意义**：
*   `--L`: 接入点 (AP) 的数量。
*   `--N`: 每个 AP 配置的天线数量。
*   `--K`: 单天线用户的数量。
*   `--area_size`: 仿真区域的边长大小（单位：km）。
*   `--bandwidth`: 系统的运行带宽（单位：Hz）。
*   `--noise_psd`: 接收机噪声的功率谱密度（单位：dBm/Hz）。
*   `--epochs`: 训练的总轮数。
*   `--batches_per_epoch`: 每个 Epoch 内包含的训练批次数量。
*   `--batch_size`: 每次参数更新所使用的样本批量大小。
*   `--lr`: 优化器的初始学习率。
*   `--hidden_dim`: 神经网络中特征提升层和 Transformer 的隐藏层维度。

### 4. 期待的输出
通过在命令行中运行该文件，终端将依次输出以下内容：

## 文件: `joint_qat.py`

### 1. 计划步骤映射

该文件实现了【科研计划】中的 **步骤 6: 联合量化感知训练 (Joint Quantization-Aware Training)**。

**服务于该步骤的方式**：
- **端到端集成**：将前期设计的自适应量化器（`AdaptiveQuantizer`）与预训练的全精度图神经网络（`QAT_FPGNN`）无缝集成，构建了端到端的联合量化感知模型 `JointQATModel`。
- **位宽感知与特征融合**：在 GNN 的输入端，除了接收量化后的复数软信息和局部 SNR 外，还拼接了策略网络输出的期望位宽（`expected_bits`），使得 GNN 能够感知各 AP 的量化精度并自适应调整注意力权重。
- **联合损失函数与退火策略**：在训练过程中，损失函数不仅包含信号恢复的 MSE Loss，还引入了对平均传输位宽（目标为 2.5 bits）的二次惩罚项。同时，采用了 Gumbel-Softmax 的温度参数（Tau）退火策略，使网络在训练初期具有探索性，后期逐渐趋于确定性的硬判决选择。

### 2. 总体功能描述

该文件是整个科研项目中的核心训练脚本，主要用于在动态带宽约束下，联合优化 AP 端的“累进式位宽策略网络”与 CPU 端的“异构 GNN 检测器”。

**核心逻辑与扮演的角色**：
- **预训练权重继承与适配**：`QAT_FPGNN` 类继承了全精度 GNN 的网络结构，并通过 `load_pretrained` 方法加载预训练权重。为了适配新增的位宽特征（`expected_bits`），代码对第一层特征提升网络（`feature_mlp`）的权重维度进行了平滑扩展（新增维度初始化为 0），确保了预训练能力的无损迁移。
- **联合前向传播**：在 `JointQATModel` 中，输入的软信息和 SNR 首先通过量化器得到离散的量化值和期望位宽。随后，这些特征被拼接并送入 GNN 进行多用户干扰消除（MU-IC）和空间分集合并。整个过程利用直通估计器（STE）保持了梯度的可导性。
- **在线数据生成与评估**：脚本在训练循环中动态生成海量不同发射功率下的信道数据，以增强模型的泛化能力。训练完成后，在严格分离的测试集上，对比了传统全精度均值合并（Dist-Full）与所提 QAT-GNN 模型的 BER 性能，验证了在极低平均回传开销（约 2.5 bits）下的性能增益。

### 3. 命令行调用方式

```bash
python joint_qat.py \
    --L 16 \
    --N 4 \
    --K 8 \
    --area_size 1.0 \
    --bandwidth 20000000.0 \
    --noise_psd -174.0 \
    --epochs 50 \
    --batches_per_epoch 100 \
    --batch_size 512 \
    --lr 0.001 \
    --hidden_dim 64 \
    --pretrained_model fp_gnn_model.pth
```

**参数物理意义**：
- `--L`: 系统中接入点（AP）的总数量。
- `--N`: 每个接入点（AP）配备的天线数量。
- `--K`: 系统中单天线用户的总数量。
- `--area_size`: 仿真区域的边长大小（单位：km）。
- `--bandwidth`: 系统的运行带宽（单位：Hz）。
- `--noise_psd`: 接收机底噪的功率谱密度（单位：dBm/Hz）。
- `--epochs`: 联合量化感知训练的总轮数。
- `--batches_per_epoch`: 每个训练 Epoch 中包含的批次（Batch）数量。
- `--batch_size`: 每次参数更新所使用的样本数量。
- `--lr`: 优化器的初始学习率。
- `--hidden_dim`: GNN 特征提升层和 Transformer 编码器的隐藏层特征维度。
- `--pretrained_model`: 步骤 3 中训练好的全精度 GNN 模型权重文件的路径。
