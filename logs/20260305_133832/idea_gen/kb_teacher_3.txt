*   **Near-Field Propagation:** Because the array is so large, users/scatterers are often within the "Rayleigh distance," meaning the signal reaches the array as a spherical wave rather than a plane wave. This requires a **polar-domain** representation (angle + distance).
*   **Spatial Non-Stationarity:** Not every antenna "sees" every scatterer. Each scatterer has a specific **Visibility Region (VR)** on the array. This creates a "clustered sparsity" where certain sub-arrays are active for a path while others are not.

### 2. Core Methodology
The authors propose a framework based on **Alternating Maximum A Posteriori (MAP)** estimation, which breaks the joint problem into three iterative modules:

#### A. The 2D Markov Prior Model
To capture the fact that if one sub-array "sees" a scatterer, its neighbors likely do too, the authors use a **2D Markov chain** to model the VR. This provides a structured prior that encourages the detection of clustered, contiguous visibility regions rather than random, isolated active antennas.

#### B. The Three-Module Alternating Framework
1.  **Channel Estimation Module (IF-VBI):** 
    *   Uses **Inverse-Free Variational Bayesian Inference**. 
    *   Standard Bayesian methods require inverting huge matrices, which is computationally impossible for XL-MIMO. This module uses a majorization-minimization approach to avoid matrix inversion, ensuring the algorithm is scalable.
2.  **VR Detection Module (Structured EP):** 
    *   Uses **Expectation Propagation (EP)** to detect which sub-arrays see which scatterers. 
    *   It combines observation data with the 2D Markov prior to "clean up" the VR detection, even in low Signal-to-Noise Ratio (SNR) environments.
3.  **Grid Update Module (Gradient Ascent):** 
    *   In digital grids, the true location of a scatterer rarely falls exactly on a grid point (the "off-grid" problem). 
    *   This module uses gradient ascent to refine the polar-domain grid (angle and distance) dynamically, reducing "leakage" errors and improving accuracy.

---

### 3. Key Takeaways

*   **Superior Accuracy at Low SNR:** By exploiting the 2D clustered sparsity of the VR (via the Markov prior), the proposed method significantly outperforms traditional algorithms like Orthogonal Matching Pursuit (OMP), especially when the signal is weak.
*   **Computational Efficiency:** The "Inverse-Free" (IF) nature of the VBI module is critical. It reduces the complexity of channel estimation from a cubic relationship with the number of antennas to a linear/quadratic one, making it practical for arrays with thousands of elements.
*   **Effective Near-Field Handling:** By using a polar-domain grid and a dynamic grid-refinement step, the algorithm accurately estimates both the direction and the distance of scatterers, which is essential for near-field 6G beamforming.
*   **Spatial Correlation is Key:** The paper demonstrates that treating sub-arrays as independent (as many baseline methods do) is inefficient. Exploiting the spatial correlation between adjacent sub-arrays is the most effective way to handle non-stationary XL-MIMO channels.

### Conclusion
The paper provides a robust, low-complexity solution for 6G XL-MIMO systems that enables the network to accurately "see" which parts of a massive antenna array are receiving useful signals and precisely estimate the channel parameters in the near-field.

### 1. Core Methodology

The paper addresses two primary challenges in Extra-Large Scale MIMO (XL-MIMO): the **prohibitive computational complexity** of processing thousands of antennas and **spatial non-stationarity** (where a user’s signal is only visible to a portion of the array, known as a Visibility Region or VR).

**A. Distributed Architecture**
The BS antenna array is divided into $B$ sub-arrays, each managed by a **Local Processing Unit (LPU)**. These LPUs perform complex signal processing in parallel, significantly reducing the bottleneck at the **Central Processing Unit (CPU)**.

**B. Variational Message Passing (VMP)**
Instead of performing massive matrix inversions (like Zero-Forcing), the LPUs use VMP to iteratively estimate:
*   **Transmitted symbols ($x$):** The data sent by users.
*   **Noise precision ($\lambda$):** A measure of the residual interference and noise at each sub-array.
VMP allows the system to approximate the posterior distribution of the symbols with much lower complexity (scaling linearly/quadratically rather than cubically with the number of users).

**C. Exploiting Spatial Non-Stationarity (VR-Awareness)**
The authors propose a "VR-aware" design. Since users are only partially visible to the array, the CPU doesn't need to fuse data from every LPU. They introduce four data fusion modes:
*   **PWR (Power-based):** Fuses only the sub-arrays containing the majority of a user’s energy.
*   **NOP (Noise precision-based):** Fuses sub-arrays with the highest estimated signal quality.
*   **HYB (Hybrid):** Combines both power and noise precision metrics.
*   **Normal:** Fuses all sub-arrays.

**D. Successive Interference Cancellation (SIC)**
To boost performance in crowded scenarios, the CPU employs an SIC mechanism. It uses a new **Likelihood Ratio (LR)** metric to determine the detection order, detecting the most "certain" user first, canceling their interference, and updating the remaining LPUs.
