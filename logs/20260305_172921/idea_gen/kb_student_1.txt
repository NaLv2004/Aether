*   **Prior Score:** Handled by a pre-trained SGM (like NCSNv2), which has learned the rich structure of natural images.

*   **The Problem:** SGMs operate on noise-perturbed versions of the data. Directly using a standard likelihood score causes a mismatch that leads to poor reconstruction or divergence.
*   **The Solution:** The authors derived a closed-form approximation for the likelihood score that explicitly accounts for the current noise scale ($\beta_t$) in the diffusion process.
*   **Key Assumptions:** They assume the measurement matrix $A$ is **row-orthogonal** (or approximately so, which is true for many high-dimensional Gaussian matrices). This allows them to treat each measurement as independent, leading to a tractable, element-wise computation of the score.

**3. Algorithm: Annealed Langevin Dynamics**
The final algorithm, **QCS-SGM**, iteratively updates a random initialization by combining the pre-trained SGM's prior score with their derived noise-perturbed likelihood score. ### **Core Methodology**

The ToDMA framework operates by treating wireless communication as a sequence-to-sequence problem, similar to how Large Language Models (LLMs) handle text.

1.  **Tokenization & Modulation (Transmitter):**
    *   Source signals (images, text, etc.) are converted into discrete **tokens** using pre-trained models (e.g., VQ-GAN for images).
    *   Instead of traditional bit-to-symbol mapping, each token is mapped to a codeword from a **shared orthonormal modulation codebook**.
    *   Multiple devices transmit these codewords simultaneously over the same frequency-time resources (**non-orthogonal, grant-free transmission**).

2.  **Active Token Detection (Receiver):**
    *   The receiver (Base Station) projects the overlapping signals onto the orthonormal basis to identify which tokens were transmitted across the entire "pool" of active devices.

3.  **Token Assignment via CSI:**
    *   The receiver uses **Channel State Information (CSI)** to link specific tokens to specific devices. Because each device has a unique channel "signature," most tokens can be accurately assigned to their originating source.

4.  **Collision Mitigation via Semantic Orthogonality:**
    *   **The Problem:** If two devices transmit the same token simultaneously, a collision occurs, making it impossible to distinguish which device sent which instance of that token based on CSI alone.
    *   **The Solution:** The receiver treats collided or missing tokens as **[MASK]** tokens. It then utilizes a **pre-trained bi-directional transformer** (like a Vision Transformer or BERT) to predict the missing tokens.
    *   **Semantic Orthogonality:** This is the core concept—the idea that different messages (e.g., two different images) have distinct contextual structures. The transformer uses the surrounding "context" of a device's sequence to "fill in the blanks" caused by collisions, effectively "de-mixing" the signals in the semantic domain.

*   **Significant Latency Reduction:** Traditional orthogonal access (like OFDM) requires splitting bandwidth among all potential devices, leading to high latency as the number of devices grows.
